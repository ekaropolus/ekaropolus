# Data Integration, Migration, and Operational Reporting

## Status

Past capability that still supports current work.

## What it solves

Many advanced AI or digital twin projects depend on ordinary but difficult data
work: extracting, cleaning, mapping, reconciling, migrating, integrating, and
reporting operational data. This capability comes from enterprise and public
systems work and remains relevant to current civic infrastructure.

## Technical work

- Source-to-target mapping.
- Data migration.
- Interface design.
- Operational reporting.
- SQL/HANA-style transformations.
- Batch processing.
- Data quality checks.
- Exception handling.
- Reconciliation.
- Documentation for handoff.

## Current relevance

- Source ledgers in Hadox.
- Digital twin data contracts.
- Public-data ingestion.
- Evidence extraction.
- CRM and opportunity tracking.
- Portfolio case-study generation.
- Public/private export discipline.

## Past foundations

- Large-scale interface work.
- Migration and integration projects.
