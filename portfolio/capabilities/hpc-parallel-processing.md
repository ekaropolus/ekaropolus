# HPC and Parallel Processing

## Status

Current and past capability.

## What it solves

Some digital twin and scientific intelligence workloads are too large or too
slow for naive local processing. This capability identifies which workloads
should move to parallel processing, HPC, cloud, or distributed execution.

## Technical work

- HPC adoption strategy.
- Parallel-processing learning paths.
- OpenMPI exploration.
- Batch-processing design.
- Local workstation vs VPS vs supercomputer comparison.
- Workload decomposition.
- Documentation of run modes.

## Candidate workloads

- Geospatial tiling.
