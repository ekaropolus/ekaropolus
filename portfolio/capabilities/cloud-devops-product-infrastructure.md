# Cloud, DevOps, and Product Infrastructure

## Status

Current and past capability.

## What it solves

Applied research work becomes useful only when it can run, be inspected, be
deployed, and be maintained. This capability covers the infrastructure layer
behind research platforms, prototypes, public pages, and internal tools.

## Technical work

- Repository structuring.
- Docker and container workflows.
- VPS deployment.
- GitHub and Gitea remotes.
- CI/CD-style deployment.
- Node, Python, React/Vite, Express, and related app stacks.
- Local LAN tools.
- Documentation for operations, maintenance, and handoff.

## Current applications

- Hadox and Polisplexity web surfaces.
- EVA Breast Cancer Archive Platform.
- Research Operations Ledger and research-operations infrastructure.
- Public opportunity radar.
- Internal CRM/export workflows.
- Research platform deployment.

## Past foundations

- Enterprise deployment discipline.
- Google Cloud, AWS, Azure, BigQuery, Databricks, and SAP/cloud training.
- Large-system operational constraints.
