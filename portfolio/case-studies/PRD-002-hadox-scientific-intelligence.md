# PRD-002 - Hadox Scientific Intelligence

## Public title

Scientific intelligence infrastructure for research, innovation, and
institutional decision-making.

## Public surfaces

- https://hadox.org
- https://github.com/Hadox-Research-Labs

## Role

Founder and research/product architect.

## Problem

Scientific and institutional evidence usually lives across incompatible
documents, datasets, spreadsheets, email threads, reports, proposals, public
records, and expert memory. AI can accelerate synthesis, but without source
discipline it can also destroy traceability.

## Technical approach

Hadox builds evidence-to-decision systems. The core pattern is to preserve source
lineage while creating normalized and enriched layers that can support research,
strategy, public platforms, dashboards, diligence, and AI-enabled workflows.

The operating principle is that the evidence base must outlive the report. A
proposal, dashboard, or decision memo should be generated from a structured
evidence layer that can be inspected, extended, corrected, and reused.

## Components

- Source ledgers.
- Data contracts.
- Raw, normalized, enriched, and public-export layers.
- Researcher, institution, capability, and opportunity maps.
- Grounded capability packages.
- Decision briefs.
- Venture, program, and diligence workflows.
- Public and private intelligence platforms.
- Reproducible reports and client packages.

## Pipeline pattern

```text
inputs
  -> source ledger
  -> extraction / parsing
  -> normalized entities
  -> enrichment and relationship mapping
  -> review queue
  -> generated outputs
  -> public/private export
```

Input types:

- papers;
- PDFs;
- spreadsheets;
- public registries;
- application forms;
- opportunity pages;
- validated relationship records;
- meeting evidence;
- repository documentation;
- expert notes.

Output types:

- concept notes;
- partner maps;
- opportunity radars;
- capability matrices;
- data rooms;
- public pages;
- CRM records;
- research briefs;
- executive memos;
- technical proposals.

## AI governance pattern

AI is useful only if the system keeps:

- source references;
- confidence and uncertainty;
- human review state;
- public/private sensitivity;
- claim-to-source links;
- update history.

The intended system is not "AI writes a report." It is "AI helps maintain an
evidence workspace that can generate better reports."
